## Soal

Kode berikut digunakan untuk mengambil detail pesanan dari server, menampilkan informasi pesanan pada modal, serta melakukan konfirmasi pesanan ketika tombol konfirmasi ditekan.

Meskipun kode tersebut dapat berjalan dengan baik, masih terdapat beberapa pelanggaran terhadap prinsip **Clean Code**. Lakukan refaktorisasi sehingga kode menjadi lebih terstruktur, mudah dipahami, dan mudah dipelihara.

## Kode Program

* [index.js](./index.js)

## Deskripsi

Program ini digunakan untuk mengambil data pesanan dari sebuah API berdasarkan `orderId`, kemudian menampilkan informasi pesanan pada sebuah modal dialog. Selain itu, program juga menyediakan tombol untuk menutup modal dan tombol konfirmasi pesanan apabila status pesanan belum **Delivered**.

Pada kode awal, seluruh proses mulai dari pengambilan data, manipulasi DOM, hingga pemasangan event listener dilakukan di dalam satu fungsi yang panjang. Kondisi tersebut membuat kode sulit dibaca, sulit dipelihara, dan kurang sesuai dengan prinsip **Single Responsibility Principle (SRP)**.

Oleh karena itu dilakukan refaktorisasi dengan memecah fungsi besar menjadi beberapa fungsi yang memiliki tanggung jawab masing-masing.

---

## Analisis Permasalahan

Beberapa permasalahan pada kode awal antara lain:

1. Fungsi `fetchOrderDetails()` memiliki terlalu banyak tanggung jawab, yaitu mengambil data, membuat tampilan, mengatur modal, dan mengatur event listener.
2. Kode menjadi panjang sehingga sulit dibaca dan dipelihara.
3. Event listener dipasang setiap kali fungsi dipanggil sehingga berpotensi menimbulkan duplikasi listener.
4. Sebagian logika sulit digunakan kembali pada bagian lain program karena tidak dipisahkan menjadi fungsi-fungsi kecil.

---

## Hasil Refaktorisasi

Refaktorisasi dilakukan dengan membagi fungsi menjadi beberapa bagian, yaitu:

* `createOrderDetails()` untuk membuat tampilan informasi pesanan.
* `setupConfirmButton()` untuk mengatur tombol konfirmasi.
* `showOrderModal()` untuk menampilkan modal beserta komponennya.
* `fetchOrderDetails()` hanya bertugas mengambil data dari server dan memanggil fungsi lain yang diperlukan.

Dengan pembagian tersebut, setiap fungsi hanya memiliki satu tanggung jawab sehingga kode menjadi lebih modular.

---

## Alur Program

1. Fungsi `fetchOrderDetails()` mengambil data pesanan dari API menggunakan `fetch()`.
2. Data yang diterima dikirim ke fungsi `showOrderModal()`.
3. Fungsi `showOrderModal()` menampilkan modal dan memanggil `createOrderDetails()` untuk membuat isi tampilan pesanan.
4. Fungsi `setupConfirmButton()` mengatur apakah tombol konfirmasi ditampilkan atau disembunyikan sesuai status pesanan.
5. Pengguna dapat menutup modal atau melakukan konfirmasi pesanan melalui tombol yang tersedia.

---

## Penjelasan

Refaktorisasi dilakukan tanpa mengubah perilaku program, tetapi hanya memperbaiki struktur kode agar lebih mudah dipahami.

Pemisahan fungsi berdasarkan tanggung jawab membuat setiap bagian kode memiliki tujuan yang jelas. Selain itu, penggunaan fungsi-fungsi kecil mempermudah proses debugging, pengembangan fitur baru, maupun pengujian program di masa mendatang.

Penggunaan `onclick` pada tombol juga menghindari penambahan event listener yang berulang setiap kali detail pesanan ditampilkan.

---

## Keuntungan Refaktorisasi

1. Kode menjadi lebih rapi dan mudah dibaca.
2. Setiap fungsi hanya memiliki satu tanggung jawab (Single Responsibility Principle).
3. Mengurangi kompleksitas fungsi utama.
4. Mempermudah pemeliharaan dan pengembangan aplikasi di masa mendatang.
5. Mengurangi potensi duplikasi event listener.

---

## Kesimpulan

Refaktorisasi dilakukan dengan memecah fungsi yang memiliki banyak tanggung jawab menjadi beberapa fungsi yang lebih kecil sesuai fungsinya masing-masing. Pendekatan ini membuat kode lebih modular, mudah dipahami, dan lebih sesuai dengan prinsip-prinsip Clean Code, tanpa mengubah fungsionalitas utama dari program.
