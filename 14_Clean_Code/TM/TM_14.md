## Soal

Dari dua kode berikut, manakah yang lebih ingin diperbaiki ketika terjadi masalah pada pukul 1 malam? Jelaskan alasannya.

---

## Kode Program

### Kode Pertama

```javascript
function processUser(user) {
  if (user) {
    if (user.isActive) {
      if (user.hasPermission) {
        return doSomething(user)
      }
    }
  }
  return null
}
```

### Kode Kedua

```javascript
function processUser(user) {
  if (!isValidCandidate(user)) return null;
  return doSomething(user);
}

function isValidCandidate(user) {
  return user && user.isActive && user.hasPermission;
}
```

---

## Deskripsi

Kedua kode memiliki tujuan yang sama, yaitu memeriksa apakah objek `user` memenuhi syarat tertentu sebelum menjalankan fungsi `doSomething(user)`.

Perbedaannya terletak pada cara penulisan logika validasi. Kode pertama menggunakan beberapa `if` yang bersarang (nested), sedangkan kode kedua memisahkan proses validasi ke dalam fungsi `isValidCandidate()` sehingga fungsi utama menjadi lebih singkat dan mudah dipahami.

---

## Analisis

Apabila harus mencari masalah pada pukul 1 malam, saya lebih memilih menangani **kode kedua**.

Kode kedua memiliki struktur yang lebih sederhana karena proses validasi dipisahkan ke dalam fungsi tersendiri. Dengan demikian, ketika terjadi kesalahan, saya dapat langsung memeriksa fungsi `isValidCandidate()` tanpa harus membaca seluruh isi `processUser()`.

Sebaliknya, kode pertama menggunakan beberapa kondisi `if` yang bersarang sehingga alur logikanya lebih sulit diikuti. Ketika jumlah kondisi bertambah, kode akan semakin sulit dibaca dan proses debugging menjadi lebih lama.

Selain itu, fungsi `isValidCandidate()` pada kode kedua dapat digunakan kembali di bagian lain program apabila diperlukan validasi yang sama, sehingga mengurangi duplikasi kode.

---

## Perbandingan

| Kode Pertama                          | Kode Kedua                       |
| ------------------------------------- | -------------------------------- |
| Menggunakan nested if                 | Menggunakan helper function      |
| Sulit dibaca ketika kondisi bertambah | Lebih ringkas dan mudah dipahami |
| Sulit dipelihara                      | Mudah dipelihara                 |
| Kurang modular                        | Lebih modular dan reusable       |
| Debugging lebih sulit                 | Debugging lebih mudah            |

---

## Penjelasan

Menurut saya, kode kedua lebih sesuai dengan prinsip **Clean Code**, terutama prinsip **Single Responsibility Principle** dan **Readability**. Fungsi `processUser()` hanya bertugas mengatur alur program, sedangkan proses validasi dipisahkan ke fungsi lain yang memiliki tanggung jawab khusus.

Struktur seperti ini membuat programmer lebih mudah memahami kode, melakukan debugging, maupun melakukan perubahan di masa mendatang tanpa memengaruhi bagian lain program.

Pemisahan logika validasi ke dalam fungsi `isValidCandidate()` membuat proses debugging menjadi lebih cepat dan mengurangi kemungkinan kesalahan saat melakukan perubahan pada kode.
