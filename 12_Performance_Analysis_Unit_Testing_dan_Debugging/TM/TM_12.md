## Soal

Diberikan sebuah fungsi `tambahPengitung()` yang digunakan untuk melakukan penjumlahan pada sebuah counter beserta file pengujiannya (`hitung.test.js`).

Analisis apakah kode tersebut sudah benar atau masih terdapat bagian yang perlu diperbaiki, kemudian jelaskan alasan perbaikannya.

---

## Kode Program

* [hitung.js](./hitung.js)
* [hitung.test.js](./hitung.test.js)

---

## Deskripsi

Program ini merupakan implementasi sederhana dari fungsi penjumlahan sebuah penghitung (counter). Fungsi menerima dua parameter, yaitu nilai penghitung saat ini (`terkini`) dan nilai penambahan (`jumlah`), kemudian mengembalikan hasil penjumlahannya.

Selain fungsi utama, program juga dilengkapi dengan unit test menggunakan modul `node:test` dan `assert` untuk memastikan fungsi bekerja sesuai dengan yang diharapkan.

Pada kode yang diberikan terdapat kekurangan, yaitu fungsi belum diekspor sehingga tidak dapat diimpor oleh file pengujian. Oleh karena itu dilakukan perbaikan agar proses testing dapat berjalan dengan baik.

---

## Analisis Kode

Bagian yang kurang tepat terdapat pada file `hitung.js`, yaitu fungsi belum diekspor sehingga tidak dapat digunakan oleh file `hitung.test.js`.

Kode sebelum perbaikan:

```javascript
function tambahPengitung(terkini, jumlah) {
  terkini = terkini + jumlah;
  return terkini;
}
```

Walaupun logika penjumlahannya sudah benar, fungsi tersebut tidak memiliki `export`, sehingga ketika dijalankan bersama kode berikut:

```javascript
import { tambahPengitung } from './hitung.js';
```

akan menyebabkan proses import gagal.

---

## Perbaikan Program

Perbaikan dilakukan dengan menambahkan `export` pada fungsi serta menyederhanakan proses penjumlahan.

Kode setelah perbaikan:

```javascript
function tambahPengitung(terkini, jumlah) {
  return terkini + jumlah;
}

export { tambahPengitung };
```

Dengan perubahan tersebut, fungsi dapat digunakan oleh file pengujian dan tetap menghasilkan output yang sama.

---

## Alur Program

1. Fungsi `tambahPengitung()` menerima nilai penghitung saat ini dan nilai penambahan.
2. Fungsi menjumlahkan kedua nilai tersebut.
3. Hasil penjumlahan dikembalikan ke pemanggil fungsi.
4. File `hitung.test.js` menjalankan beberapa skenario pengujian menggunakan `assert.strictEqual()`.
5. Jika hasil sesuai dengan nilai yang diharapkan, maka seluruh test dinyatakan berhasil.

---

## Output

Seluruh unit test berhasil dijalankan.

```text
✔ 5 tambah 3 sama dengan 8
```

## Penjelasan

1. Pengujian pertama memastikan bahwa `5 + 3` menghasilkan `8`.
2. Pengujian kedua memastikan bahwa `0 + 10` menghasilkan `10`.
3. Fungsi penjumlahan bekerja dengan benar setelah dapat diimpor melalui `export`.
4. Penulisan `return terkini + jumlah;` membuat kode lebih ringkas tanpa mengubah hasil perhitungan.

---

## Kesimpulan

1. Logika penjumlahan pada fungsi sudah benar dan menghasilkan nilai yang sesuai.
2. Kekurangan pada kode awal adalah tidak adanya `export`, sehingga fungsi tidak dapat diimpor oleh file pengujian.
3. Penambahan `export` membuat fungsi dapat digunakan oleh modul lain sesuai konsep modular JavaScript.
4. Unit testing berhasil membuktikan bahwa fungsi bekerja sesuai dengan hasil yang diharapkan.
